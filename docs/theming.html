Theming

Customize colors, typography, spacing, and more using CSS custom properties.

How Theming Works

Amris uses a three-layer token architecture. Each layer builds on the one below it:

  1. Primitive tokens — raw values (hex colors, rem sizes, pixel values). These never change between themes.
  2. Semantic tokens — meaningful names (--am-text, --am-primary) that map to primitives. Light theme is the default.
  3. Dark overrides — the dark theme remaps semantic tokens to different primitives (e.g., --am-text changes from near-black to near-white).

Components only reference semantic tokens, so switching themes is automatic. You can also override any token at any level to create your own brand theme.

Light and Dark Mode

Set the theme on <am-theme-provider>:

<!-- Follow OS preference -->
<am-theme-provider theme="system">...</am-theme-provider>

<!-- Force light -->
<am-theme-provider theme="light">...</am-theme-provider>

<!-- Force dark -->
<am-theme-provider theme="dark">...</am-theme-provider>

Toggle dynamically with JavaScript:

const provider = document.querySelector('am-theme-provider');
provider.theme = 'dark';  // 'light' | 'dark' | 'system'

Customizing the Brand Palette

Override the primary color scale to change the brand identity across all components. Set these on the theme provider or any ancestor element: 

<am-theme-provider style="
  --am-color-primary-50:  #eff6ff;
  --am-color-primary-100: #dbeafe;
  --am-color-primary-200: #bfdbfe;
  --am-color-primary-300: #93c5fd;
  --am-color-primary-400: #60a5fa;
  --am-color-primary-500: #3b82f6;
  --am-color-primary-600: #2563eb;
  --am-color-primary-700: #1d4ed8;
  --am-color-primary-800: #1e40af;
  --am-color-primary-900: #1e3a8a;
  --am-color-primary-950: #172554;
">
  <!-- All components now use blue as the primary color -->
</am-theme-provider>

Per-Component Overrides

Many components expose their own CSS custom properties for fine-grained control. These are documented with @cssprop in each component's source: 

<!-- Override a specific component's radius -->
<am-button style="--am-button-radius: 9999px">Pill Button</am-button>

<!-- Override all inputs in a section -->
<div style="--am-input-radius: 0">
  <am-input label="Square corners"></am-input>
</div>

Styling with ::part

Every component exposes CSS parts for deeper styling without piercing the shadow DOM: 

<style>
  /* Style the native button inside am-button */
  am-button::part(button) {
    text-transform: uppercase;
    letter-spacing: 0.05em;
  }

  /* Style the input element inside am-input */
  am-input::part(input) {
    font-family: 'Georgia', serif;
  }

  /* Style the dialog panel */
  am-dialog::part(dialog) {
    border: 2px solid var(--am-primary);
  }
</style>
