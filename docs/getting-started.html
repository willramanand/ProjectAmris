Getting Started

Add Amris to your project and start using components in minutes.

Installation

Install the package from npm:

npm install amris

Setup

Wrap your application in <am-theme-provider>. This element provides all design tokens (colors, spacing, typography, etc.) as CSS custom properties that cascade into every Amris component. 

<!-- index.html -->
<script type="module" src="node_modules/amris/dist/amris.js"></script>

<am-theme-provider theme="system">
  <!-- Your app content goes here -->
</am-theme-provider>

The theme attribute controls the color scheme:

  • "system" — follows the user's OS preference (default)
  • "light" — forces light mode
  • "dark" — forces dark mode

Using Components

Plain HTML

Import the full bundle and use any component as a custom element. No build step required. 

<script type="module">
  import 'amris';
</script>

<am-theme-provider>
  <am-button variant="primary">Save Changes</am-button>
  <am-input label="Email address" type="email"></am-input>
  <am-badge variant="success">Active</am-badge>
</am-theme-provider>
Save Changes Active

ES Module Imports

For tree-shaking, import only the components you need. Each component self-registers its custom element tag on import. 

// Import specific components
import 'amris/core';                         // foundations + common components
import { AmButton } from 'amris';            // individual class

// Or import everything
import 'amris';

Framework Usage

Amris components are standard web components. They work in any framework that supports custom elements. 

<!-- Svelte -->
<script>
  import 'amris';
  let name = '';
</script>

<am-theme-provider>
  <am-input label="Name" on:am-input={(e) => name = e.detail.value}></am-input>
  <am-button variant="primary" on:click={handleSave}>Save</am-button>
</am-theme-provider>
// React (with ref or wrapper)
import 'amris';

function App() {
  return (
    <am-theme-provider>
      <am-button variant="primary">Save</am-button>
    </am-theme-provider>
  );
}

Core vs Full Bundle

Amris ships two entry points:

  • amris — the full bundle with all 50+ components
  • amris/core — a smaller bundle with foundations, actions, layout, and common form inputs

Use the core bundle for smaller apps, then switch to the full bundle as you need more components.

<!doctype html>
<html lang="en">

    <meta charset="UTF-8" />

            Sign In
          </am-button>
        </am-card>
      </am-surface>
    </am-theme-provider>
  </body>

Browser Support

Amris targets modern evergreen browsers. It requires native support for:

  • Custom Elements v1
  • Shadow DOM v1
  • CSS Custom Properties
  • ES Modules
  • ElementInternals (for form-associated components)

This means Chrome/Edge 90+, Firefox 90+, and Safari 16.4+.
